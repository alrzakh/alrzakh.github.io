# Personal Website Documentation

This repository contains the source code for my personal website hosted on GitHub Pages. The website serves as a portfolio, blog, and a platform to share my projects and interests.

## Project Structure

- **_config.yml**: Configuration settings for the Jekyll site, including site title and description.
- **index.md**: The main landing page of the website.
- **404.html**: Custom 404 error page for non-existent pages.
- **README.md**: Documentation about the project.
- **LICENSE**: Licensing terms for the project.
- **Gemfile**: Specifies Ruby gems required for the Jekyll site.
- **.gitignore**: Lists files and directories to be ignored by Git.
- **.gitattributes**: Defines attributes for path-based settings in the repository.
- **.editorconfig**: Maintains consistent coding styles across different editors.
- **.vscode/**: Contains settings and configurations for Visual Studio Code.
- **_layouts/**: Contains layout templates for the site.
- **_includes/**: Contains reusable HTML snippets for the site.
- **_sass/**: Contains Sass files for styling.
- **assets/**: Contains CSS and JavaScript files for the website.
- **_data/**: Contains YAML files for navigation, social links, and projects.
- **_posts/**: Contains blog posts written in Markdown format.
- **_drafts/**: Contains draft blog posts.
- **pages/**: Contains additional pages like About, Contact, Projects, Blog, and Uses.
- **CNAME**: Specifies a custom domain for the GitHub Pages site.
- **robots.txt**: Provides instructions to web crawlers.

## Getting Started

To get started with this project, follow these steps:

1. **Clone the Repository**: 
   ```
   git clone https://github.com/yourusername/personal-website-github-pages.git
   ```

2. **Navigate to the Project Directory**:
   ```
   cd personal-website-github-pages
   ```

3. **Install Dependencies**:
   Make sure you have Ruby and Bundler installed, then run:
   ```
   bundle install
   ```

4. **Run the Jekyll Server**:
   ```
   bundle exec jekyll serve
   ```
   Your site will be available at `http://localhost:4000`.

## Customization

You can customize the website by editing the following files:

- **_config.yml**: Update the site title, description, and other metadata.
- **index.md**: Modify the content of the landing page.
- **_posts/**: Add new blog posts by creating Markdown files in this directory.
- **pages/**: Create or edit additional pages like About, Contact, and Projects.
