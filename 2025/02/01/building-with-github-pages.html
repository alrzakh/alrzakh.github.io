Building With Github Pages

Posted on February 01, 2025 by

Getting Started

To begin, you need to create a GitHub account if you don’t already have one. Once you have an account, you can create a new repository for your website. Make sure to name it in the format username.github.io, where username is your GitHub username. This naming convention is crucial as it allows GitHub to recognize it as a Pages site.

Setting Up Your Site

  1. Choose a Framework: While GitHub Pages supports static HTML, using a static site generator like Jekyll can simplify the process. Jekyll allows you to write your content in Markdown, which is then converted to HTML.

  2. Create Your Files: After setting up Jekyll, you’ll have a basic structure with folders for layouts, includes, and posts. Customize these files to fit your style and needs.

  3. Configuration: Edit the _config.yml file to set your site title, description, and other metadata. This file is essential for configuring your Jekyll site.

Customizing Your Website

  • Layouts and Includes: Modify the layout files in the _layouts directory to change the overall look of your site. Use the _includes directory for reusable components like headers and footers.

  • Styling: Use Sass to style your website. The _sass folder allows you to create variables and mixins for consistent styling across your site.

  • JavaScript Functionality: Add interactivity with JavaScript. You can include scripts in the assets/js directory to enhance user experience.

Deploying Your Site

Once you’re satisfied with your website, it’s time to deploy it. Push your changes to the GitHub repository, and your site will be live at https://username.github.io. You can also set up a custom domain by adding a CNAME file to your repository.

Conclusion

Building a personal website with GitHub Pages is a rewarding experience. It not only showcases your skills but also serves as a platform for sharing your thoughts and projects. With the right tools and a bit of creativity, you can create a stunning online presence. Happy coding!
